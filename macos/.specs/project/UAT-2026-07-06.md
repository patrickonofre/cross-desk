# Roteiro UAT consolidado — T38 + T47 + T28 (sessão única, 2 macs)

Cobre os três UATs pendentes numa sessão só: **T38** (discovery-pairing), **T47** (file-transfer E1) e **T28** (input-polish + latência). Ordem pensada para minimizar retrabalho: parear primeiro (A), depois arquivos (B), por último a matriz de sleep/wake (C — mexe no estado do sistema).

**Convenção:** SRV = mac servidor (teclado/mouse físicos), CLI = mac cliente. Marcar ✅/❌ + observação em cada item; ao final, registrar resultados nas seções "Registro" (aqui mesmo) e sintetizar no STATE.

## 0. Setup (15 min)

- [ ] Build novo na máquina de build: `macos/Scripts/make-app.sh` (151 testes verdes antes: `cd macos/CrossDeskKit && swift test`).
- [ ] Distribuir `macos/build/CrossDesk.app` para a outra máquina (zip → AirDrop/ssh). Na máquina de destino: `xattr -cr /caminho/CrossDesk.app` (sem notarização ainda).
- [ ] TCC: SRV precisa **Monitoramento de Entrada**; CLI precisa **Acessibilidade**. Se o toggle aparecer ligado mas falhar (lição beta.1→2): `tccutil reset All dev.crossdesk.mac` + conceder de novo.
- [ ] Config limpa para testar pareamento do zero: apagar `~/Library/Application Support/CrossDesk/config.json` nas DUAS máquinas (guarda cópia se quiser preservar algo).
- [ ] Mesma LAN, mDNS funcionando (rede doméstica normal serve).

## A. T38 — discovery-pairing (aceitações 1–8 do spec)

- [ ] **A1 visibilidade:** SRV inicia como servidor → aparece na lista do CLI em ≤3 s. SRV para → some da lista em ≤10 s.
- [ ] **A2 pareamento feliz:** CLI toca "Parear…", digita o token exibido no SRV → conecta; selo "Pareado" nos dois lados.
- [ ] **A3 token errado:** digitar token inválido → erro claro no CLI (timeout de handshake), sem travar; corrigir e conectar.
- [ ] **A4 rotação persistida:** após A2, `config.json` dos DOIS lados tem `pairedSecret` (32 hex). Reiniciar os dois apps → reconexão silenciosa SEM digitar token (fingerprint igual nos logs).
- [ ] **A5 esquecer no servidor:** SRV "Esquecer pareamento" → token novo exibido; CLI tenta conectar → falha com o segredo antigo e (R30) cai para pedir token; parear de novo com o token novo.
- [ ] **A6 esquecer no cliente:** CLI "Esquecer" → volta à lista; parear de novo.
- [ ] **A7 Rede Local negada→concedida (incerteza 2):** revogar Rede Local do CLI (Ajustes → Privacidade → Rede Local) → lista vazia + aviso laranja aparece; conceder de volta → lista volta sem reiniciar o app (se precisar reiniciar, registrar).
- [ ] **A8 fallback IP:** com descoberta funcionando, conectar mesmo assim por "Conectar por IP…" usando endereço mostrado no painel do SRV → funciona igual.
- [ ] **A-extra (incerteza 4):** se as duas máquinas tiverem o mesmo nome de dispositivo, registrar se o Bonjour renomeia ("Nome (2)").

## B. T47 — file-transfer E1 (aceitações 1, 2, 4, 7, 8 — 3/5/6 já automatizadas)

Pré: pareado e conectado (bloco A). Cursor pode estar em qualquer máquina; ⌘C/⌘V normais.

- [ ] **B1a pequeno SRV→CLI:** copiar (⌘C) um arquivo pequeno no Finder do SRV → painel do CLI mostra "Recebendo/Arquivos prontos" → ⌘V numa pasta do Finder do CLI → arquivo íntegro (abrir).
- [ ] **B1b pequeno CLI→SRV:** mesmo caminho inverso.
- [ ] **B2 grande (≥1 GB):** gerar no SRV: `mkfile -n 1g /tmp/teste-1gb.bin` (ou `dd if=/dev/urandom of=/tmp/teste-1gb.bin bs=1m count=1024`). Copiar → CLI mostra **oferta pendente** (>200 MB): "1 arquivo (1,07 GB) — Receber" → tocar "Receber" → barra de progresso avança → termina em `~/Downloads/CrossDesk/` + Finder revela. Depois: repetir e tocar **Cancelar** no meio → sem arquivo parcial em Downloads nem staging (`~/Library/Caches/CrossDesk/incoming/` limpo), erro/estado claro no painel.
- [ ] **B4 rede caída no meio:** iniciar transferência do 1 GB → desligar Wi-Fi do SRV no meio → CLI mostra falha limpa, sem parcial visível; religar Wi-Fi → input reconecta sozinho (heartbeat); nova transferência funciona.
- [ ] **B7 input fluido durante transferência:** durante o 1 GB, mover mouse/digitar na máquina remota → sem stutter perceptível. Se degradar: registrar (mitigação `serviceClass = .background`, design D5).
- [ ] **B8 build antigo:** (se houver beta.9 guardada) rodar beta.9 num lado e build novo no outro → input funciona, transferência simplesmente não acontece, sem crash/log de erro em loop.
- [ ] **B-extra race da rotação:** logo após um pareamento NOVO (A5), copiar arquivo nos primeiros ~2 s → pode falhar 1× (PSK em rotação, race conhecida) → repetir cópia → funciona. Registrar se aconteceu.

## C. T28 — input-polish (aceitações + medições + decisões de design)

- [ ] **C1 invisibilidade:** com cursor no CLI, a seta do SRV some de verdade (não só parada). Voltar → seta reaparece no ponto certo. (Inverso: seta do CLI some quando o cursor está no SRV — toggle "Esconder o cursor" ligado.)
- [ ] **C2 matriz standby/wake:** com trava ativa, rodar em cada máquina: `pmset displaysleepnow` → acordar → trava/ocultação continuam (watchdog + reassert). Repetir com screensaver e com troca de resolução/monitor se disponível. Dock/Mission Control não "solta" a seta.
- [ ] **C3 sleep-survival (incerteza 4):** `cd macos/CrossDeskKit && swift run conceal-spike --sleep-test` → registrar se a dissociação cai no wake (esperado: cai → R19 justificado).
- [ ] **C4 scroll/momentum (incerteza 3):** trackpad no SRV controlando CLI: Safari/Xcode fazem rubber-band/inércia natural com as fases repassadas?
- [ ] **C5 autorepeat + modificadores:** segurar tecla (repete no CLI), atalhos ⌘C/⌘Tab/⇧ funcionam; Esc 3× SEMPRE devolve o controle (inclusive com rede caída — desligar Wi-Fi e testar).
- [ ] **C6 coalescing (R23):** mexer o mouse rápido (trackpad 90–120 Hz) e ler no log `metrics [server] capturedMouseMove=…` ao parar: fila>1 em regime? Sim → plugar `MoveCoalescer` no drain (vira task); não → registrar "não necessário".
- [ ] **C7 latência p95:** ler p95 captura→injeção das métricas (`logSummary` no stop) → registrar número (fecha R14 do kvm-mvp).
- [ ] **C8 regressão kvm-mvp:** travessia de borda nas duas direções, clique, drag de janela, borda de retorno.

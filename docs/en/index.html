How it works

One server, one or more clients.

One Mac is the server (owns the keyboard and mouse); the others are clients. The cursor crosses the screen edge as if it were a single desktop.

  1. 1On the server, pick the screen edge — CrossDesk shows the pairing code and address.
  2. 2On the client, enter the address and short code — pairing derives the key locally, nothing travels over the network.
  3. 3Move the cursor to the edge: control switches to the other Mac. Esc×3 always returns control to the server.

Privacy and security

Nothing travels in plaintext.

CrossDesk connects your Macs over UDP with DTLS-PSK encryption — no plaintext mode, no account, no cloud server. The protocol is public and documented.

  • No user account
  • No in-app telemetry
  • Open, versioned protocol in the repository

Installation

Not notarized by Apple yet — that's expected.

CrossDesk doesn't have an Apple Developer account yet, so macOS blocks the first launch as a precaution. Allow it like this:

  1. 1

    Download and move to Applications

    Download the latest release .zip and move CrossDesk.app to your Applications folder.

  2. 2

    Remove quarantine

    Open Terminal and run: xattr -cr /Applications/CrossDesk.app

  3. 3

    Grant permissions

    In System Settings → Privacy & Security: Input Monitoring on the server, Accessibility on the client.

After this first install, new versions arrive through the app itself via Sparkle — no more browser trips.

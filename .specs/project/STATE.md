# STATE — CrossDesk (raiz)

## Decisões

- **2026-07-03 — ADR-002: apps nativas separadas por OS** (decisão do usuário, supera ADR-001/Rust). macOS primeiro, Swift 100% nativo, pasta `macos/`. Protocolo neutro versionado + golden vectors como contrato entre apps. Racional em [TECH-DECISION.md](TECH-DECISION.md).
- **2026-07-03 — Protocolo próprio**, sem compatibilidade Synergy/Barrier na v1.
- **2026-07-03 — Keycodes em USB HID Usage IDs** no fio (neutro entre CGKeyCode/scancode/evdev).
- **2026-07-03 — Geometria independente por máquina** (protocolo §5): resolução/layout nunca trafegam; cliente detecta a própria borda de retorno (LEAVE_REQUEST 0x12). Vale para todas as implementações (Windows/Linux herdam o contrato).

- **2026-07-04 — file-transfer antecipada** (decisão do usuário; era "ideia adiada"/não-objetivo v1). Staged: E1 clipboard de arquivos (⌘C/⌘V, API pública), E2 drag real só se spike S1 provar (Synergy falhou nisso; Deskflow removeu). Destino fallback: `~/Downloads/CrossDesk/`. Specs: `macos/.specs/features/file-transfer/`.
- **2026-07-05 — PROTOCOL.md §2 ganhou regra normativa de truncagem de campos `utf8`** (achado em code review profundo da app mac): cortar string longa por contagem de bytes crus pode partir um escalar Unicode ao meio, e o receptor descarta a mensagem inteira como corrompida. Vale para toda implementação (Windows/Linux herdam a regra) — ver Changelog do protocolo e `macos/.specs/project/STATE.md` (lições) para o detalhe técnico e o fix de referência (`WireStrings.utf8Prefix`).
- **2026-07-06 — token curto de pareamento reduzido de 8 para 6 chars** (decisão do usuário; PROTOCOL.md §1, `XXXX-XXXX` → `XXX-XXX`, ≈39 → ≈29 bits). Troca deliberada de UX (menos dígitos pra ler/digitar) por menos entropia; aceitável porque o token nunca trafega no fio (deriva PSK localmente nos dois lados) e é substituído pelo segredo rotacionado de 128 bits logo no 1º handshake (R29) — a única janela de exposição é o handshake de pareamento em si, já documentada. Sem mudança de formato de fio nem golden vectors. Vale para toda implementação (Windows/Linux herdam a regra).
- **2026-07-07 — release-1.0 especificada** (repo já é público em `github.com/patrickonofre/cross-desk` desde 2026-07-03, com as 15 tags `v0.1.0-beta.*` já publicadas). Decisões do usuário (ver `.specs/features/release-1.0/context.md`): (1) descartar só as tags beta local+remoto, histórico de commits intacto; (2) 1.0 só fecha depois dos 5 UATs pendentes passarem (discovery-pairing T38, file-transfer T47, input-polish T28, layout-ux T57, autostart T61 — todos em `macos/.specs/`); (3) distribuição ad-hoc assinado + `xattr -cr` documentado, sem Apple Developer ID por ora; (4) licença MIT. Spec+tasks em `.specs/features/release-1.0/`.
- **2026-07-07 — CrossDesk 1.0.0 publicado**: os 5 UATs fechados (usuário reportou passando, sem bugs — ver `macos/.specs/project/STATE.md`), versão bumpada (`CFBundleShortVersionString` 1.0.0, build 17), `LICENSE`/`CHANGELOG.md`/README prontos, as 15 tags `v0.1.0-beta.*` deletadas (local+remoto — `git ls-remote --tags origin` só mostra `v1.0.0`), tag anotada `v1.0.0` e [GitHub Release](https://github.com/patrickonofre/cross-desk/releases/tag/v1.0.0) publicados com o build universal (`CrossDesk.zip`) anexado. Commits: `8b4a71f` (autostart), `f21390c` (docs/UATs), `74e1b02` (LICENSE/CHANGELOG/README), `bf1936a` (bump versão). Achado após o fato: apagar a tag não apaga o Release do GitHub (objetos separados) — os 15 releases `v0.1.0-beta.*` ficaram órfãos apontando pra tag inexistente; apagados também (`gh release delete`, confirmado pelo usuário) — `gh release list` só mostra `v1.0.0`. **Fase 1 do ROADMAP (app macOS) fechada.**
- **2026-07-07 — update-check especificado** (pós-1.0; feature de referência era `mac-metrics-view`, que usa Sparkle completo). Decisão do usuário: **não** replicar Sparkle aqui — checagem via GitHub Releases API (`releases/latest`), aviso passivo no popover (label + botão "Baixar" pra página da release), sem auto-instalar. Motivo: cross-desk é SPM puro sem Xcode project (embutir Sparkle.framework+XPC exigiria script manual de sign) e já tem incidente documentado de troca de bundle quebrando TCC — replicar Sparkle repetiria essa classe de risco sem necessidade. Antecipa parte do item "Auto-update por plataforma" da Fase 4 (ROADMAP); auto-instalação de verdade fica pra depois, se decidido. Spec+tasks em `macos/.specs/features/update-check/`.
- **2026-07-07 — update-check implementado e publicado como v1.1.0** (T62-T64 completos: `UpdateChecker` + testes, wiring em `AppState`/`AppConfig`, UI em `MenuBarView` — 190/190 testes verdes). Decisão do usuário: republicar a 1.0 estava descartado (sobrescrever a tag `v1.0.0` não daria a feature pra quem já instalou — código novo não chega em app já rodando — e reescrever tag pública já publicada é destrutivo) — em vez disso, bump normal de versão (SemVer minor, feature aditiva). Commits: `b69393a` (feat), `0841498` (chore bump 1.1.0/build 18). - **2026-07-07 — v1.1.1 publicado (patch)**: 2 fixes do hotkey oculto do debug-console (detalhe em `macos/.specs/project/STATE.md` — silêncio de falha de registro + troca de combinação por colisão com ferramentas de dev). SemVer patch (bug fix, sem feature nova, sem mudança de protocolo). Commits: `6c2172d`, `5f86a9b` (fixes), `46af740` (bump 1.1.1/build 19). Tag `v1.1.1` + [GitHub Release](https://github.com/patrickonofre/cross-desk/releases/tag/v1.1.1) publicados (zip universal, `CrossDesk Dev` signing). CHANGELOG/notas do Release ficaram deliberadamente vagas ("ajustes internos de estabilidade") — debug-console é feature oculta (R45: "nenhuma menção fora do código-fonte e desta spec"), citar o atalho num Release público quebraria isso. - **2026-07-07 — v1.1.2 publicado (patch)**: usuário reportou "abrir ao iniciar não funcionou" (autostart, R50-R54). Causa raiz não era bug no toggle — era App Translocation: abrir o zip baixado sem `xattr -cr` roda o app de um path efêmero, e `SMAppService.mainApp.register()` gravava esse path, que some no próximo login (entrada fica "ligada" em Ajustes mas nunca abre). Já tinha sido flagado como achado colateral em 2026-07-06 (ver `macos/.specs/project/STATE.md`) mas só virou "reforçar no roteiro", nunca fix de código — desta vez virou: `LoginItem.isTranslocated` novo detecta o path, `AppState.toggleLoginItem()` recusa agir nesse caso, UI (`MenuBarView`) desabilita o toggle com aviso pedindo pra mover pra Aplicativos (ou remover quarentena) e reabrir. Notarização real continua pendência aberta (linha 26) — isto é rede de segurança contra o mesmo sintoma, não a cura estrutural. SemVer patch (bug fix, sem mudança de protocolo). Commits: `efc8adc` (fix), `e5233f7` (bump 1.1.2/build 20). Tag `v1.1.2` + [GitHub Release](https://github.com/patrickonofre/cross-desk/releases/tag/v1.1.2) publicados (zip universal, `CrossDesk Dev` signing).
